# HTTP Request Capture

The NLM CLI includes built-in HTTP request/response capture functionality for debugging and comparing API calls.

## Overview

HTTP requests can be captured during test runs and exported to human-readable [txtar](https://pkg.go.dev/golang.org/x/tools/txtar) format for easy inspection.

## Quick Start

### Basic Usage

Run tests with HTTP recording enabled:

```bash
# Record HTTP traffic during tests (requires credentials)
go test ./cmd/nlm -run TestCLICommands -httprecord=. -v

# Export recordings to txtar format
go test ./cmd/nlm -run TestCLICommands -export-txtar -v
```

### View Captured Requests

```bash
# View the txtar file
cat internal/api/testdata/TestName.txtar
```

## Txtar Format

The txtar format is a simple text-based archive format that's easy to read and diff:

```
HTTP Request/Response Trace
Source: testdata/test.httprr
Generated by: github.com/tmc/nlm/internal/httprr

-- 01_AHyHrd_request.http --
POST /batchexecute?rpcids=AHyHrd HTTP/1.1
Host: notebooklm.google.com
Content-Type: application/x-www-form-urlencoded
Cookie: [REDACTED]

f.req=[["AHyHrd",["notebook-id","instructions"]]]

-- 01_AHyHrd_response.http --
HTTP/1.1 200 OK
Content-Type: text/plain

[response body]
```

## Security

By default, sensitive information is automatically redacted:
- `Cookie` headers → `[REDACTED]`
- `Authorization` headers → `[REDACTED]`

To include secrets (for local debugging only):

```go
rr.ExportToTxtar("output.txtar", true) // includeSecrets=true
```

**Warning:** Never commit txtar files with secrets enabled!

## Use Cases

### 1. Compare CLI vs Browser Requests

```bash
# Capture CLI request
go test -run TestAudioCreate -httprecord=. -export-txtar

# Compare with browser request from DevTools
# Copy browser curl → format as txtar → diff
```

### 2. Debug API Issues

```bash
# Capture failing request
go test -run TestFailingCommand -httprecord=. -export-txtar

# Inspect the exact request/response
cat internal/api/testdata/TestFailingCommand_01_RPCId_request.http
```

### 3. Document API Behavior

```bash
# Record successful API calls
go test -run TestAllCommands -httprecord=. -export-txtar

# Commit the txtar files as documentation
git add internal/api/testdata/*.txtar
```

## Programmatic Usage

### Export from Code

```go
import "github.com/tmc/nlm/internal/httprr"

// Open existing httprr recording
rr := &httprr.RecordReplay{}
rr.SetFile("testdata/recording.httprr")

// Export to txtar
err := rr.ExportToTxtar("output.txtar", false) // false = redact secrets
if err != nil {
    log.Fatal(err)
}
```

### Custom RPC ID Extraction

The txtar exporter automatically extracts RPC IDs from NotebookLM requests for better file naming:

```
f.req=[["AHyHrd", ...]] → 01_AHyHrd_request.http
f.req=[["VUsiyb", ...]] → 02_VUsiyb_response.http
```

## Test Flags

| Flag | Description |
|------|-------------|
| `-httprecord=PATTERN` | Record HTTP traffic for tests matching pattern |
| `-export-txtar` | Export httprr recordings to txtar after tests |
| `-httprecord-debug` | Enable detailed recording debug output |
| `-httpdebug` | Log all HTTP requests/responses |

## Examples

### Record and Export Single Test

```bash
go test ./cmd/nlm \
  -run TestCLICommands/audio_commands.txt \
  -httprecord=. \
  -export-txtar \
  -v
```

### Record All Tests

```bash
go test ./cmd/nlm \
  -run TestCLICommands \
  -httprecord=. \
  -export-txtar \
  -v
```

### Debug Specific RPC

```bash
# Enable all debug output
go test ./cmd/nlm \
  -run TestAudioCreate \
  -httprecord=. \
  -httprecord-debug \
  -httpdebug \
  -export-txtar \
  -v
```

## File Locations

- **HTTP recordings**: `internal/api/testdata/*.httprr`
- **Txtar exports**: `internal/api/testdata/*.txtar`
- **Test scripts**: `cmd/nlm/testdata/*.txt`

## Integration with Scripts

The scripttest framework automatically exports HTTP recordings when `-export-txtar` is enabled:

```bash
# Run scripttest with export
go test ./cmd/nlm -export-txtar -v

# All httprr files are automatically converted to txtar
ls internal/api/testdata/*.txtar
```

## Troubleshooting

### No recordings found

Make sure to run with `-httprecord=.` first:

```bash
# This won't work (no recording)
go test -export-txtar

# This works (record then export)
go test -httprecord=. -export-txtar
```

### Secrets in output

By default, secrets are redacted. If you see actual secrets:
1. Check that `includeSecrets=false` (default)
2. Verify the redaction is working
3. **Never commit files with secrets!**

### Export fails

Check file permissions and paths:

```bash
# Ensure testdata directory exists
mkdir -p internal/api/testdata

# Check file permissions
ls -la internal/api/testdata/*.httprr
```
